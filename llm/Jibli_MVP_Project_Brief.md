# Jibli — MVP Project Brief

## Project Overview

Jibli is a mobile marketplace that connects travelers leaving Tunisia with people abroad who want items from Tunisia.

Travelers can use their extra luggage space to bring requested items and earn a reward, while requesters get access to Tunisian products that are hard to find abroad.

The platform focuses on simple coordination, trust, and communication between travelers and requesters.

The first version will be a mobile app MVP designed to validate the core idea and user demand.

---

## Core Concept

The application works through a simple interaction:

1. A traveler posts an upcoming trip (e.g., Tunis → Paris).
2. A requester sees the trip and submits an item request.
3. The traveler accepts the request.
4. Both users communicate through in-app chat.
5. The traveler delivers the item when arriving.
6. The requester confirms delivery and leaves a rating.

This creates a peer-to-peer delivery network powered by travelers.

---

## Target Users

### Travelers

Tunisians traveling from Tunisia to another country who have extra luggage space and want to earn money by bringing items.

### Requesters

Tunisians living abroad who want products from Tunisia such as food, clothes, or small items.

- Strong accent color (red/orange)
- Card-based listings
- Large images for listings
- Rounded UI elements
- Minimal typography
- Bottom navigation bar

The interface should feel modern, simple, and fast, similar to marketplace or booking apps.

**Primary design goals:**

- Clarity
- Fast navigation
- Easy posting

---

## Main Navigation Structure

The application will use bottom tab navigation.

**Main sections:**

- Home
- Trips
- Requests
- Messages
- Profile

This keeps the navigation simple and familiar for users.

## Core Features (MVP)

### User Authentication

Users must be able to:

- Register an account
- Log in
- Create a basic profile

**Profile includes:**

- Name
- Profile picture
- Location
- Rating (after transactions)

**Purpose:** establish trust between users.

### Trip Posting

Travelers can create a trip listing.

**Trip information includes:**

- Departure city
- Destination city
- Travel date
- Available luggage capacity
- Optional note

**Example:**

> Tunis → Paris
> June 10
> 5kg available

Trips appear in the main feed.

### Trip Browsing

Users can browse available trips.

Each trip is displayed as a card showing:

- Traveler profile
- Route
- Travel date
- Rating
- Available space

Users can open the trip to see full details.

### Item Request

Users can request an item from a traveler.

**Request information includes:**

- Item name
- Description
- Estimated weight
- Reward amount
- Optional photo

**Example request:**

> Makroudh
> 1kg
> Reward: 20€

Requests are sent to the traveler.

### Request Management

Travelers can view incoming requests and choose to:

- Accept the request
- Reject the request

Accepted requests become active orders.

### Messaging System

Users must be able to chat directly after a request is accepted.

Chat is used to coordinate:

- Item purchase details
- Meeting location
- Delivery time

This communication happens inside the app.

### Order Status Tracking

Orders follow a simple lifecycle:

1. Requested
2. Accepted
3. Bought
4. Delivered
5. Completed

This helps both users track progress.

### Ratings and Reviews

After delivery, requesters can rate the traveler.

**Rating includes:**

- Star rating
- Optional comment

Ratings are shown on traveler profiles to build trust and reputation.

### User Activity Dashboard

Users can see their activity in one place.

**Sections include:**

- My Trips
- My Requests
- My Orders

This helps users track everything easily.

### Notifications

The app will send notifications for key actions:

- New request received
- Request accepted
- New message
- Delivery confirmation

Notifications help keep users engaged and informed.

---

## Features Excluded From MVP

The following features will not be included in the first version:

- In-app payment system
- Escrow payment protection
- Identity verification
- Luggage tracking
- AI-based matching
- Advanced admin moderation

These features can be introduced after the product is validated.

---

## MVP Goal

The goal of the MVP is to validate three key assumptions:

1. Travelers are willing to post trips.
2. Requesters are willing to request items.
3. Both users are comfortable coordinating deliveries through the app.

If these behaviors occur consistently, the product can evolve into a full marketplace platform.

---

## Product Vision

Jibli aims to become the main platform connecting the Tunisian diaspora with travelers, making it easy to receive products from Tunisia anywhere in the world.

The long-term vision is to build a trusted global delivery network powered by travelers.
